### Programming guessing game
ในบทนี้จะเป็นการเรียนรู้พื้นฐานของ rust ด้วยการทำโปรเจกต์จริง โดยจะเรียนรู้แนวคิดเกี่ยวกับ let, match, method, function, ไลบรารีภายนอก และอีกมากมาย และจะได้เรียนรู้แบบลงลึกในบทถัด ๆ ไป โดยในบทนี้จะได้ฝึกฝนพื้นฐานเท่านั้น

โดยโจทย์สำหรับข้อนี้ โปรแรกมจะสุ่มเลจระหว่าง 1 ถึง 100 จากนั้นจะให้ผู้เล่นป้อนตัวเลขที่ทาย และโปรแกรมจะแจ้งว่าตัวเลขน้อยเกินหรือมากเกินไป หากทายถูกโปรแกรมจะแสดงข้อความจบการทำงาน

2 บรรทัดใช้สำหรับแสดงข้อความเกี่ยวกับเกมขึ้นมาบนจอ

    let mut guess = String::new();

เราสามารถสร้างตัวแปรในโปรแกรมได้ด้วยการใช้คำสั่ง `let` ตัวอย่างเพิ่มเติม

    let apples = 5;

โดยในภาษา rust **โปรแกรมจะไม่สามารถแก้ไขข้อมูลได้ในภายหลัง** (เป็นค่าเริ่มต้น) โดยหากต้องการให้โปรแกรมสามารถเปลี่ยนแปรงข้อมูลได้ จะต้องเติมคำว่า `mut` ไว้ด้านหน้าชื่อตัวแปร

    let apples = 5; // immutable (เปลี่ยนแปรงไม่ได้)
    let mut bananas = 5; // mutable (เปลี่ยนแปรงได้)

เครื่องหมาย `=` เป็นการกำหนดข้อมูลให้กับตัวแปร

    let mut guess = String::new();

การเรียกใช้ `String::new()` จะสร้างสิ่งที่เรียกว่าอินสแตนซ์ใหม่ของ String ขึ้นมา โดย `String` เป็นชนิดข้อมูลที่อยู่ในไลบรารี `std` ซึ่งสามารถจัดเก็บข้อความได้ในรูปแบบ UTF-8

ไวยากรณ์ `::` (path separator) ในบรรทัด `::new()` จะใช้สำหรับเข้าถึงของที่อยู่ด้านใน type, module, namespace โดยจากตัวอย่าง `String::new()` หมายถึง เรียกใช้งานฟังก์ชัน `new()` ที่อยู่ใน type String

โดยสรุปได้ว่า `let mut guess = String::new();` หมายถึง สร้างตัวแปรที่สามารถแก้ไขข้อมูลข้างในได้ โดยข้อมูลข้างในจัดเก็บเป็น `String`

#### การรับข้อมูลจากผู้ใช้

    io::stdin()
        .read_line(&mut guess)

หากเราไม่ได้นำเข้า module io ด้วยคำสั่ง `use std::io` ในตอนต้นของโปรแกรม เราสามารถเรียกใช้ฟังก์ชันได้โดยการเรียกฟังก์ชันในรูปแบบ `std::io::stdio`

`.read_line(&mut guess)` เป็นการเรียกใช้งานเมธอด read_line เพื่อรับข้อมูลจากผู้ใช้ โดยข้อมูลที่ผู้ใช้ใส่มาจะถูกนำไปเก็บไว้ที่ guess ผ่านคำสั่ง `&mut guess` (เรียกว่าอาร์กิวเมนต์)

โดยการใช้งาน `read_line` จะเป็นการนำข้อมูลที่ผู้ใช้ใส่เข้ามา มาต่อกับ string เดิมที่มีอยู่แล้ว (ไม่เขียนทับเนื้อหาเดิม)

เครื่องหมาย `&` บอกว่าอาร์กิวเมนต์นี้เป็นตัวอ้างอิง (reference) ซึ่งช่วยให้โปรแกรมของเราจากส่วนต่าง ๆ สามารถเข้าถึงข้อมูลชิ้นเดียวกันได้ โดยไม่ต้องคัดลอกข้อมูลนั้นลงในหน่วยความจำหลาย ๆ ครั้ง

refernece จะมีทั้งหมด 2 แบบ คือ
- **immutable** ไม่สามารถเปลี่ยนแปรงได้ เช่น &guess
- **mutable** สามารถเปลี่ยนแปรงได้ เช่น &mut guess

#### เพิ่มเติม
ความแตกต่างระหว่าง :: และ . คือ
- `::` ใช้เข้าถึงของที่อยู่ภายใน type, module, enum โดยถูกเรียกว่า `associated items`
- `.` ใช้เข้าถึงของที่อยู่ภายใน instance โดยใช้เข้าถึง method, field

#### การรับมือข้อผิดพลาด

    .expect("Failed to read line");


ใน `read_line` ทุกอย่างที่ผู้ใช้พิมพ์ จะถูกใส่ลงไปใน string ที่เราส่งให้มัน แต่ในขณะเดียวกัน มันจะคืนค่าเป็นชนิด `Result` กลับมาด้วย โดย `Result` เป็นชนิดข้อมูแบบ enumaration (enum) ซึ่งเป็นชนิดข้อมูลที่สามารถอยู่ได้ในหลายสถานะ โดยแต่ละสถานะจะเรียกว่า `variant`

`Result` ที่ `read_line` ตอบกลับมาจะมี 2 variant คือ
- `Ok` หมายถึง การดำเนินการสำเร็จ
- `Err` หมายถึง การดำเนินการล้มเหลว (มักจะมีข้อมูลเกี่ยวกับวิธีหรือสาเหตุที่การดำเนินการล้มเหลว)

โดยประเภท `Result` จะมีเมธอดที่กำหนดไว้ชื่อว่า `expect` โดยหาก `Result` มีค่าเป็น `Err` เมธอดนี้จะทำงานทันทีและหยุดการทำงานของโปรแกรม

#### การแสดงข้อมูลออกมาผ่าน println!

    println!("You guessed: {guess}");

ชุดคำสั่งดังกล่าว จะพิมพ์ข้อความที่มีข้อมูลอยู่แล้ว โดย `{}` จะใช้เป็น **ตัวแทนค่า (placeholder)** โดยเมื่อเราต้องการนำข้อมูลจากตัวแปรอะไรไปใส่ไว้ในข้อความ เราสามารถใช้ `{}` ได้

แต่หากเราใช้งาน `{}` เปล่า ๆ โดยไม่มีการระบุตัวแปรลงไป ฟังก์ชันจะทำการดึงข้อมูลมาใส่ตามลำดับ

    let x = 5;
    let y = 10;

    println!("x = {x} and y + 2 = {}", y + 2); // x = 5 and y + 2 = 12

การระบุเวอร์ชันของ crate เราสามารถระบุเลขเวอร์ชันตรง ๆ เช่น `0.8.5` หรือสามารถใช้งานเครื่องหมาย `SemVer` เพื่อระบุว่าเวอร์ชันที่จะใช้ จะต้องมากกว่าเวอร์ชันที่ระบุ เช่น `^0.8.5` หมายถึง เวอร์ชันที่จะใช้จะต้องมากกว่า 0.8.5 *แต่ต่ำกว่า 0.9.0*

#### ทำไมถึงน้อยกว่า 0.9.0 ?
เนื่องจากการใช้งาน `^` จะรับประกันว่าเราจะได้รับแพทช์ล่าสุดของไลบรารี และจะยังคงสามารถใช้งานได้ดี (มีการอัพเวอร์ชันแค่เลขหลักที่ 3) แต่หากเวอร์ชันเลขหลักที่ 2 ถูกแพทย์ จะไม่รับประกันว่าชุดคำสั่งของไลบรารีที่เรากำลังใช้งานจะยังสามารถใช้งานได้อยู่

หลังจากที่เราระบุไลบรารีที่เราต้องการใช้ลงใน `Cargo.toml` แล้ว เราสามารถใช้งานคำสั่ง `cargo build` ได้ทันที โดยระบบจะติดตั้งไลบรารีที่เราระบุ และไลบรารีต่าง ๆ ที่โปรไลบรารีของเราต้องการใช้งาน

`Ordering` เป็น enum อีกชนิดหนึ่งที่มี variat คือ `Less`, `Greater`, `Equal` ซึ่งเป็นผลลัพธ์สามอย่างที่เป็นไปได้เมื่อเปรียบเทียบค่าสองค่า

    match guess.cmp(&secret_number) {
        Ordering::Less => println!("Too small!"),
        Ordering::Greater => println!("Too big!"),
        Ordering::Equal => println!("You win!")
    }

ฟังก์ชัน `cmp` (ย่อมาจาก compare) ทำหน้าที่เปรียบเทียบค่าสองค่า และสามารถเรียกใช้กับอะไรก็ได้มาเปรียบเทียบกัน

โดย `cmp` จะรับค่าแบบ `reference` ของสิ่งที่เราต้องการนำมาเปรียบเทียบด้วย ซึ่งในที่นี้ คือการทำค่า guess (ตัวเลขที่ผู้ใช้ทาย) มาเทียบกับ secret_number (เลขที่สุ่มมา)

หลังจากเปรียบเทียบกันแล้ว ฟังก์ชัน `cmp` จะส่งคืนมาเป็น variant ของ enum ที่ชื่อว่า `Ordering` จากนั้นเราจะใช้สิ่งที่เรียกว่า `match` expression เพื่อตัดสินใจว่าจะทำอะไรต่อ โดยดูจาก `cmp` ที่ส่งกลับมา

`match` expression จะประกอบไปปด้วยส่วนที่เรียกว่า Arms (เงื่อนไขต่าง ๆ) ซึ่งจะประกอบด้วย
1. Pattern (รูปแบบ): เอาไว้ตรวจสอบว่าค่าตรงกับรูปแบบที่กำหนดหรือไม่
2. Code: ชุดคำสั่งที่จะทำงานเมื่อรูปแบบตรงกับ arm ดังกล่าว

ในภาษา rust, `match` เป็นเครื่องมือที่ดีมาก โดยมันจะบังคับให้เราเขียนดักจับสถาณการณ์ต่าง ๆ ที่โปรแรกมอาจจะเจอได้อย่างครอบคลุม และบังคับให้เราต้องจัดการกับทุกกรณีที่เป็นไปได้ (ป้องกันข้อผิดพลาด)


    match guess.cmp(&secret_number) { // เรียก cmp เพื่อเทียบค่า
        Ordering::Less => println!("Too small!"),  // Arm ที่ 1: ถ้าค่าน้อยกว่า
        Ordering::Greater => println!("Too big!"), // Arm ที่ 2: ถ้าค่ามากกว่า
        Ordering::Equal => println!("You win!"),   // Arm ที่ 3: ถ้าค่าเท่ากัน
    }

หากเราเขียนไม่ครอบคลุม จะไม่สามารถ*คอมไพล์*ได้

    let guess: u32 = guess.trim().parse().expect("Please type a number!")

rust อนุญาติให้เราสร้างชื่อตัวแปรซ้ำได้ โดยมักใช้ในการแปลงชนิดข้อมูลจากประเภทหนึ่งเป็นอีกประเภทหนึ่ง
โดยจากชุดคำสั่งด้านบน สามารถอธิบายได้ดังนี้
- `guess.trim()` ทำการตัดช่องว่างซ้ายขวาและ `\n` ออกจากข้อความ (การกด enter จะทำให้ได้ค่า `\n` มาด้วย)
- `.parse()` พยายามแปลง string ให้เป็นตัวเลข ซึ่งฟังก์ชันนี้ฉลาดพอที่จะรู้ว่าเราต้องการแปลงเป็นประเภทอะไร (ดูจากชนิดข้อมูลของตัวแปร)
- `.expect()` ดักจับข้อผิดพลาดที่อาจเกิดขึ้น

เครื่องหมาย `:` หน้าชื่อตัวแปรหมาย เป็นการกำหนดชนิดข้อมูลที่ตัวแปรสามารถเก็บได้ เช่น `u32` หมายถึง เลขจำนวนเต็ม 32 บิต

    let guess: u32 = guess.trim().parse() {
            Ok(num) => num,
            Err(_) => continue,
        };

เนื่องจาก `parse` จะส่งคืนค่าประเภท `Result` ซึ่งจะมี `Ok` และ `Err` ให้เราจัดการ
โดยเราสามารถใช้งานร่วมกับ `match` ได้ จากชุดคำสั่งด้านบน
- หากสามารถแปลงเป็นตัวเลขได้ จะได้รับ `Result` เป็น `Ok` จะทำการส่งคืนข้อมูล
- หากไม่สามารถแปลงเป็นตัวเลขได้ จะได้รับ `Result` เป็น `Err` จะทำการข้ามการวนซ้ำไปรอบถัดไป

    use std::io;
    use rand::Rng;
    use std::cmp::Ordering;

    fn main() {
        println!("Guess the number!");

        let secret_number = rand::thread_rng().gen_range(1..=100);

        loop {
            println!("Please input your guess.");
            let mut guess = String::new();

            io::stdin()
                .read_line(&mut guess)
                .expect("Failed to read line");

            println!("You guessed: {guess}");
            println!("The secret number is: {}", secret_number);

            let guess: u32 = match guess.trim().parse() {
                Ok(num) => num,
                Err(_) => continue,
            };

            match guess.cmp(&secret_number) {
                Ordering::Less => println!("Too small!"),
                Ordering::Greater => println!("Too big!"),
                Ordering::Equal => {
                    println!("You win!");
                    break;
                }
            }
        }
    }
