### Cargo
เป็นระบบสร้างและจัดการแพ็กเกจของ rust โดยเราจะใช้ในการจัดการโปรเจ็กต์ เช่น 
- ดาวน์โหลดไลบรารีที่จำเป็น
- สร้างไลบรารีและส่งออกให้นักพัฒนาคนอื่นใช้

*ไลบรารี่ที่ระบบของเราต้องการ จะเรียกว่า dependency*

ตรวจสอบว่า cargo ถูกติดตั้งแล้วหรือยังด้วยคำสั่ง <mark>cargo --version</mark>

#### สร้างโปรเจคด้วย cargo
เราสามารถใช้งานคำสั่ง <mark>cargo new <project_name></mark>

โดยปกติการสร้างโปรเจค หากสร้างไว้ในโครงสร้างที่มี .gitignore อยู่แล้ว ไฟล์ .gitignore ของ rust จะไม่ถูกสร้างขึ้นด้วย เราสามารถใช้งานคำสั่ง <mark>cargo new --vcs=git</mark>เพื่อให้ระบบสร้าง .gitignore ของ rust ให้ด้วย

#### TOML (Tom's Obvious Minima; Language)
ไฟล์ดังกล่าว (Cargo.toml) จะใช้สำหรับตั้งค่าโปรเจค โดยโครงสร้างของไฟล์มีดังนี้

    [package]

เป็นหัวข้อของแพ็กเกจ โดยเราสามารถตั้งค่าโปนเขตได้จากบรรทัดถัด ๆ ไปได้

    name = "hello_cargo"
    version = "0.1.0"
    edition = "2024"

บรรทัดดังกล่าว เป็นการกำหนดการตั้งค่าโปรเจ็ก โดยมีการระบุ
- ชื่อโปรเจก
- เวอร์ชันของโปรเจก
- เวอร์ชันของ rust ที่ใช้งานในโปรเจค
_

    [dependencies]

ใช้สำหรับระบุไลบรารีที่จำเป็นสำหรับโปรเจ็กต์ของเรา โดยในภาษา rust แพ็กเกจเหล่านี้จะถูกเรียกว่า <mark>crates</mark>

ใน Cargo ชุดคำสั่ง rust ทั้งหมดจะต้องอยู่ใน src เสมอ ส่วนไดเร็กทอรี่ระดับบนสุดของโปรเจค จะใช้สำหรับจัดเก็บไฟล์ readme, configuration, และสิ่งอื่น ๆ ที่ไม่เกี่ยวข้องกับโค้ดของเรา ซึ่งการทำแบบนี้จะช่วยให้โปรเจ็กของเรามีความเป็นระบเบียบมากขึ้น

โดยหากเราต้องการสร้าง Cargo.toml อย่างเดียว เราสามารถใช้งานคำสั่ง <mark>cargo init</mark>

<marl>cargo build</mark> คำสั่งดังกล่าว ใช้สำหรับแปรงชุดคำสั่งทั้งโปรเจค จากภาษา rust เป็นภาษาเครื่อง (.exe) โดยจะสร้างไฟล์ไว้ใน target\debug\hello_cargo.exe

โดยการ build ระบบจะสร้างไฟล์ที่ชื่อว่า Cargo.lock โดยไฟล์นี้จะบันทึกเวอร์ชันที่แน่นอนของไลบรารีต่าง ๆ ในโปรเจ็กต์ของเรา

เราสามารถรันชุดคำสั่งที่ถูก build แล้วได้ด้วยคำสั่ง <mark>cargo run</mark>

โดยหากชุดคำสั่งของเรามีการแก้ไข ระบบจะทำการ compile ให้ก่อนจึงจะเรียกใช้งาน

เราสามารถตรวจสอบโปรเจคก่อนรันจริงได้ว่าสามารถทำงานได้หรือไม่ ผ่านคำสั่ง <mark>cargo check</mark> โดยการใช้งานคำสั่งดังกล่าว จะเร็วกว่าการใช้งาน cargo build เพราะระบบจะไม่ compile ชุดคำสั่งของเรา โดยเรามักจะใช้ *ตรวจสอบโปรเจคของเราเป็รระยะ ๆ ว่ามีข้อผิดพลาดหรือไม่*

#### Release
เมื่อโปรเจคของเราพร้อมใช้งานจริงแล้ว เราสามารถใช้งานคำสั่ง <mark>cargo build --release</mark> ได้ โดยระบบจะทำการปรับปรุงประสิทธิภาพของโปรแกรมให้เร็วที่สุด (แต่ในขั้นตอนการ build จะนานขึ้น) โดยโปรเจ็กต์ที่สร้างเสร็จแล้ว จะถูกสร้างไว้ใน target/release
