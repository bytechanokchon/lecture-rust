### Variable and Mutebility
ตัวแปรในภาษา rust โดยค่าเริ่มต้นจะไม่สามารถเปลี่ยนแปรงข้อมูลที่ตัวแปรเก็บอยู่ได้

    let x = 5;
    println!("The value of x is: {x}");
    x = 6;
    println!("The value of x is: {x}");

โดยหากเราต้องการให้ตัวแปรสามารถเปลี่ยนแปลงข้อมูลได้ เราสามารถเติมคำว่า `mut` ไว้หน้าชื่ิอตัวแปร โดยมันยังมีข้อดีอีกอย่างคือ *ทำให้ผู้อื่นรู้ว่าตัวแปรนี้จะมีการเปลี่ยนแปรงข้อมูลจากส่วนอื่น ๆ ของคำสั่ง*

    let mut y = 5;
    println!("The value of x is: {y}");
    y = 6;
    println!("The value of x is: {y}");

#### การประกาศค่าคงที่ (Declaring Constants)
ค่าคงที่คือตัวแปรที่ไม่สามารถเปลี่ยนแปลงข้อมูลได้ โดยมีความแตกต่างจากตัวแปรปกติดังนี้
- ไม่อนุญาตให้ใช้ `mut` กับตัวแปรค่าคงที่
- ไม่สามารถเก็บข้อมูลที่มาจากการคำนวนระหว่างโปรแรกมทำงานได้ เช่น const z = x + y;
- การตั้งชื่อค่าคงที่จะต้องเป็นตัวใหญ่แบบ snack case เสมอ เช่น TREE_HOURS_IN_SECONDS

โดยการใช้งานค่าคงที่ เราจะใช้คีย์เวิร์ด `const` ในการประกาศตัวแปรแทน `let`
โดยกรณีที่เราต้องใช้ `const` อาจเป็นการ จัดเก็บคะแนนสูงสุดของผู้เล่น, ความเร็วของแสง

#### Shadowing
เป็นการประกาศตัวแปรโดยใช้ชื่อเดิมที่เคยประกาศไว้อยู่แล้ว เช่น

    let x = 5;

    // Shadowing ครั้งที่ 1: เพิ่มค่าให้ x
    let x = x + 1;

    {
        // Shadowing ครั้งที่ 2: อยู่ใน Inner Scope
        let x = x * 2;
        println!("The value of x in the inner scope is: {x}"); // ผลลัพธ์: 12
    }

    println!("The value of x is: {x}"); // ผลลัพธ์: 6 (กลับมาที่ค่าก่อนเข้า Scope)

จากใน Inner scope จะพบว่า เมื่อมีการเรียก `x` มันจะไม่ได้ไปดึงข้อมูลจาก `x` แรกมาใช้ แต่จะดึงข้อมูลจาก `x` ที่สอง เนื่องจาก `x` แรกถูกบดบังจาก `x` ใหม่ที่ถูกประกาศไปแล้ว สิ่งนี้เรียกว่า shadowing

โดยการใช้งาน shadowing มีข้อดีคือทำให้เราไม่ต้องคิดชื่อตัวแปรใหม่ ๆ ในกรณีต้องทำงานกับตัวแปรเดิม ๆ
อย่างไรก็ตาม หากตัวแปรเป็น `mut` เราจะไม่สามารถใช้งาน shadowing ได้

ตัวอย่างการใช้งาน หากเรารับข้อมูลอายุของผู้ใช้ ข้อมูลที่รับเข้ามาจะเป็นรูปแบบ `string` เราจำเป็นต้องแปรงเป็น `int` ทำให้อาจต้องสร้าง 2 ตัวแปรคือ age_str และ age_int ซึ่งการทำ `shadowing` ทำให้เราใช้ตัวแปรเดิมได้เลย ไม่ต้องประกาศตัวแปรใหม่
